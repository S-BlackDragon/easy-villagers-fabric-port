Siempre verificar el plan en `plan-easy.txt` antes de hacer cualquier cambio.

**Why:** El usuario definió un flujo estricto de 3 fases para no introducir bugs acumulados. Quiere probar cada fase antes de avanzar.

**How to apply:**
- Antes de escribir código, releer la fase actual del plan
- Al terminar una fase: decirle al usuario exactamente qué probar y correr Minecraft
- Esperar confirmación del usuario antes de empezar la siguiente fase
- No saltarse ni combinar fases
